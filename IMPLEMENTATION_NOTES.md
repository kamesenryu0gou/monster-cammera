# 珍獣カメラ 実装メモ

## 概要

このプロジェクトは、縦型デジタルサイネージ向けの AI フォト体験 UI として構築しています。現在の構成では、メニュー画面から BGM 設定とパスワード認証を行い、その後に中央正方形の撮影画面へ遷移します。撮影後は、受付番号を先に発行したうえで AI 加工処理を実行し、完成画像と受付番号を表示した後、自動で撮影画面へ復帰します。

## 実装済みの主要構成

| 項目 | 内容 |
|---|---|
| フロントエンド | React + Tailwind による縦型サイネージ UI |
| バックエンド | Express + tRPC による受付番号発行、画像保存、AI 加工処理 |
| 保存対象 | 元画像、AI 加工画像、A5 2×4 の8面印刷プレビュー、受付番号、人数、割当バリアント、処理時刻 |
| AI 加工 | サーバー側で Forge ImageService を呼び出し、共通プロンプト + 5 種の派生プロンプトで変換 |
| 人数対応 | 最大 5 名まで。6 名以上は再撮影案内 |
| 番号形式 | `YYYYMMDD-####` |
| BGM | `珍獣カメラ.mp3` を使用した常時再生トグル |

## 現在の画面フロー

1. メニュー画面で BGM ON/OFF を設定し、パスワードを入力します。
2. 撮影画面では中央の正方形プレビューと撮影ボタンを表示します。
3. 3→2→1 のカウントダウンでは、数字が進むごとに文字サイズが大きくなります。
4. 撮影直後に受付番号を発行し、「AI加工中です」と受付番号を表示します。
5. AI 変換後、A5 2×4 の8面印刷プレビュー・受付番号・案内文「受け取りは店内スタッフまでお声掛けください！」を表示します。
6. 10 秒後に撮影画面へ自動復帰します。

## AI バリアント

| コード | 名称 |
|---|---|
| A | Fluffy Beast |
| B | Little Dragon |
| C | Forest Creature |
| D | Starry Night |
| E | Candy Imp |

同時に複数人を検出した場合は、写っている人数分だけ上記バリアントを重複なしで割り当てます。

## 必要な設定とランタイム前提

| 項目 | 用途 |
|---|---|
| `DATABASE_URL` | 受付番号の日次連番、ジョブ情報、保存メタデータの永続化 |
| `BUILT_IN_FORGE_API_KEY` / `BUILT_IN_FORGE_API_URL` | サーバー側の AI 画像変換処理 |
| `JWT_SECRET` | テンプレート標準の認証セッション管理 |
| ストレージ設定（テンプレート標準） | 元画像と AI 加工画像の保存 |

このプロジェクトは Node ベースの Web アプリとして構築しており、AI 加工は `server/_core/imageGeneration.ts` からサーバー側で Forge の `images.v1.ImageService/GenerateImage` を呼び出して実行します。被写体人数の解析は `server/_core/llm.ts` 経由で行い、割り当て済みの派生プロンプトを統合して 1 枚の完成画像を生成します。カメラ撮影はブラウザ権限に依存するため、実機サイネージ運用時には対象ブラウザでカメラ利用を常時許可してください。

## 現時点の運用メモ

現在のメニュー認証パスワードは仮値として `1234` をデフォルト実装しています。実運用前に秘密情報として差し替えることを推奨します。

BGM は Web プロジェクトのストレージ配下にアップロード済みで、UI から ON/OFF 切替できます。ブラウザの自動再生制限があるため、実際の再生開始は最初のユーザー操作後に安定します。

保存処理は、撮影直後に元画像をストレージへ保存し、その後の AI 生成結果を `generateImage()` 内で自動保存します。さらに最新更新では、生成済み画像から A5 2×4 の8面印刷プレビュー SVG を作成して追加保存するため、元画像・生成画像・印刷レイアウトの 3 種が自動保存対象です。

## 動作確認メモ

ブラウザプレビュー上で、メニュー画面の表示と Web preview 導線の確認を行っています。プレビュー環境ではカメラ権限未許可のため、撮影ステージでは「ブラウザのカメラ利用を許可してください。」という案内表示になる想定です。これは実機以外でのフォールバックです。

## テスト

Vitest で以下を確認しています。

| テスト対象 | 内容 |
|---|---|
| 受付番号補助ロジック | 日付キー生成とゼロ埋め |
| プロンプト割当 | 最大 5 名までの重複なし割当 |
| 変換プロンプト構築 | 被写体ごとの個別指示が最終プロンプトへ反映されること |
| 画像生成の自動保存 | Forge 生成結果がストレージへ即時保存されること |
| ログアウト既存テスト | 既存認証ルーターの回帰確認 |
